# Web 后台样式与排版总结

说明：

- 本文基于文件夹 `样式素材图` 内 8 张参考图整理，同时结合丽滋现有规范中强调的 `Premium Clinical` 方向，提炼出适合丽滋卡尔后台产品的视觉风格、模块结构与页面排版原则。
- 本次整理按 `丽滋高端医疗UI设计规范.md` 的章节架构重组，但保留 `layout.md` 原有的观察结论、排版规律与页面模板建议。
- 因此本文重点是样式素材图的结构化总结，不替代丽滋正式设计规范。

## 1. 风格定位

### 1.1 样式素材图总体定位

- 这组素材整体属于典型企业级 Web 后台、平台型产品或 SaaS 型后台界面。
- 页面主目标是：
  - 信息承载
  - 模块分区
  - 后台效率
  - 工作台与驾驶舱组织
- 不是营销页风格，也不是内容媒体风格。

### 1.2 样式素材图总体观察

- 大多数页面采用浅色底，主背景为白色或极浅灰蓝色。
- 主色集中在蓝色系，常见为亮蓝、浅蓝、蓝紫渐变、青蓝辅助色。
- 卡片容器大量使用白底、细描边、弱阴影、较大圆角。
- 页面信息密度高，但通过留白、分区卡片和固定节奏保持清晰。
- 交互风格偏企业级、平台型、SaaS 型，不是营销页风格。
- 多数界面采用“左导航 + 顶部工具栏 + 主内容区”的后台标准骨架。

### 1.3 综合风格结论

- 可提炼出的主风格关键词：
  - 专业
  - 可信
  - 高效
  - 清爽
  - 有秩序
  - 轻科技
  - 平台化
- 整体最适合被概括为：

`浅色高级企业后台 + 轻科技蓝色体系 + 卡片化模块布局 + 工作台 / 驾驶舱式信息组织`

## 2. 核心视觉语言

## 2.1 色彩系统

### 样式素材图中的色彩倾向

- 主色集中在蓝色系。
- 常见为亮蓝、浅蓝、蓝紫渐变、青蓝辅助色。
- 背景多为白色或极浅灰蓝色。
- 卡片通常为白底。

### 丽滋卡尔适配后的建议配色方向

- 主色：`#1F7AE0`
- 深导航色：`#16213D`
- 辅助青色：`#18B8C9`
- 高端点缀：`#D8C29D`
- 页面背景：`#F6F9FC`
- 卡片背景：`#FFFFFF`

### 用色规则

- 图表可在蓝、青、浅蓝、灰蓝中建立固定序列。
- 权益、会员、高端服务可少量使用砂金色。
- 避免引入紫色作为主业务色。
- 不建议直接沿用素材中的互联网运营蓝紫渐变作为丽滋后台主视觉。

## 2.2 字体与排版

### 排版气质

- 页面信息密度高，但通过留白、分区卡片和固定节奏保持清晰。
- 视觉焦点通常通过以下方式建立：
  - 页面首屏 Banner 或重点统计模块
  - 蓝色主按钮和激活态
  - 指标数字的字号和字重
  - 卡片分组与模块标题
  - 右侧任务区、消息区或待办区

### 丽滋适配后的文字表现建议

- 中文与数字统一使用 `MiSans`。
- 页面标题稳重清晰。
- 模块标题较粗。
- 正文常规字重。
- 数据数字可略高一级字重，突出经营指标和医疗关键值。

### 排版规则

- 保持后台信息密度，但不要牺牲阅读秩序。
- 重点靠模块标题、数字层级、按钮和关键卡片建立。
- 不建议使用营销化、娱乐化的大段装饰文本。

## 2.3 卡片、边框与阴影语言

### 卡片倾向

- 白底卡片系统。
- 细描边和弱阴影。
- 模块卡片大量承担信息组织任务。

### 丽滋适配建议

- 圆角推荐 `16`
- 卡片内边距推荐 `20` 或 `24`
- 边框建议使用极浅灰蓝描边
- 阴影保持轻量，避免厚重投影
- 重点卡片可以使用极浅的蓝色或青色渐变底，但不宜过强

## 3. 页面结构规范

## 3.1 整体布局

### 标准页面骨架

最常见的后台骨架为三段式：

1. 顶部栏
2. 左侧导航
3. 右侧主内容区

### 推荐结构

- 顶部栏：品牌、全局搜索、组织切换、通知、用户信息、快捷入口
- 左侧导航：一级业务入口，必要时可展开二级分组
- 内容区：面包屑、页面标题、操作区、核心模块区

### 结构结论

- `左导航 + 顶栏 + 内容区` 仍然是最适合丽滋后台的稳定骨架。
- 适合医疗服务、会员管理、预约排期、疗程记录和机构运营等复杂后台场景。

## 3.2 首页 / 工作台排版

### 工作台常见顺序

1. 页面标题与欢迎信息
2. 核心横幅或核心待办
3. KPI 统计卡
4. 业务模块卡片区
5. 快捷入口 / 常用应用
6. 消息、任务、公告、日程等补充区

### 常见布局模式

- `主内容大区 + 右侧辅助栏`
- `上方概览区 + 下方多行卡片矩阵`

## 3.3 列表页 / 广场页 / 详情页 / 驾驶舱

### 列表页 / 广场页排版

一般遵循：

1. 顶部活动或说明 Banner
2. 左侧分类筛选
3. 中部搜索与筛选栏
4. 右侧内容卡片流

适用于：

- 医生资源库
- 医美项目库
- 会员权益库
- 机构服务库
- 药械 / 商品 / 套餐列表

### 驾驶舱型页面排版

通常遵循：

1. 核心身份或机构概览
2. 关键经营指标
3. 趋势图表
4. 任务审批和提醒
5. 运营活动或排行榜

排版上常见：

- 左宽右窄的 `7:3` 或 `8:4` 分栏
- 中部大卡承载指标，右侧窄栏承载待办和动态
- 同类图表卡片保持统一高度

### 详情页 / 表单页排版

复杂业务详情页通常采用：

1. 顶部流程或标签导航
2. 基础信息表单区
3. 详情卡片区
4. 数据指标区
5. 附加资料、日志、记录区

适用于：

- 会员档案详情
- 疗程详情
- 医生排班详情
- 项目审批详情
- 机构合作信息详情

## 4. 组件风格规范

## 4.1 工作台与首页模块

### 首页工作台推荐模块

- 顶部欢迎区
- 今日待办
- 关键经营数据
- 预约与排班概览
- 会员服务提醒
- 最近审批 / 最近沟通
- 常用功能入口
- 系统公告 / 运营通知

### 工作台模块特征

- 主区用于高频业务和核心数据。
- 右侧更适合放消息、审批、日程、任务。
- 横向 Banner 可存在，但丽滋场景下应弱化互联网运营感。

## 4.2 列表页与搜索筛选模块

### 医疗后台列表页推荐模块

- 页面标题 + 主操作按钮
- 搜索栏
- 多条件筛选栏
- 数据表格
- 批量操作栏
- 分页器

### 模块规则

- 搜索与筛选是高频区，应结构稳定。
- 批量操作栏和分页应成为固定组合。
- 不建议做过多运营 Banner 或装饰区。

## 4.3 详情页与表单模块

### 医疗后台详情页推荐模块

- 基本信息卡
- 状态标签与流程节点
- 关键指标卡
- 历史记录 / 服务记录
- 附件与备注
- 底部操作栏

### 表单页排版特征

- 顶部流程或标签导航
- 基础信息表单区
- 详情卡片区
- 数据指标区
- 附加资料、日志、记录区

## 4.4 Dashboard 与 KPI 模块

### Dashboard 推荐模块

- 核心 KPI 卡
- 趋势图表
- 饼图 / 环图结构分析
- 排行榜
- 异常预警
- 待处理事项
- 日程 / 排班

### Dashboard 规则

- 指标卡优先承载核心数字。
- 图表与指标卡需要统一高度和节奏。
- 不建议做成重装饰或重暗黑驾驶舱。

## 4.5 卡片系统

### 卡片风格

- 白底卡片
- 细描边
- 弱阴影
- 较大圆角

### 使用建议

- 卡片是模块化表达的基础容器。
- 不同业务块之间通过卡片、留白、标题建立秩序。
- 不依赖重色块强切，而依赖留白和结构分区。

## 4.6 图标与插画

### 图标建议

- 图标线性化、简洁化
- 图标颜色优先跟随品牌蓝或中性灰

### 插画建议

- 插画只在首页、空状态、Banner 中少量出现
- 不建议在主业务表格和表单页中使用大量插画

## 5. 页面模块组织建议

## 5.1 工作台首页模块组织

- 顶部欢迎区
- 今日待办横卡
- 4 个核心经营指标
- 左侧主业务模块
- 右侧消息 / 审批 / 日程
- 下方常用功能入口

## 5.2 列表页模块组织

- 面包屑
- 页面标题
- 主操作按钮
- 搜索与筛选栏
- 表格
- 批量操作栏
- 分页

## 5.3 详情页与驾驶舱模块组织

### 详情页模板

- 返回 + 标题 + 状态
- 基本信息卡
- 分组表单 / 描述信息
- 过程记录
- 关联数据
- 底部操作区

### 驾驶舱模板

- 核心身份区
- KPI 数据卡
- 趋势图
- 任务提醒
- 排行榜 / 异常监控
- 公告与运营信息

## 6. 交互与可读性建议

## 6.1 信息层级与阅读节奏

- 页面信息密度高，但必须通过留白、卡片和分区维持秩序。
- 模块标题与内容之间需要明确垂直节奏。
- 不同业务块之间应保留明显间距，避免拼贴感。

## 6.2 视觉焦点组织方式

视觉焦点通常通过以下方式建立：

- 页面首屏 Banner 或重点统计模块
- 蓝色主按钮和激活态
- 指标数字的字号和字重
- 卡片分组与模块标题
- 右侧任务区、消息区或待办区

## 6.3 不建议的交互语气

- 不建议走互联网平台运营化表达。
- 不建议做大面积蓝紫渐变。
- 不建议使用过亮的蓝色发光效果。
- 不建议使用娱乐化或营销化标签样式。

## 7. 栅格与实现建议

## 7.1 页面栅格

- 桌面端建议 `12` 栅格
- 内容区统一左右留白 `24`
- 模块间距使用 `16 / 20 / 24`
- 卡片内部采用 `12 / 16 / 20` 节奏

## 7.2 常见宽度关系

- 左侧导航宽度建议 `220 - 240`
- 工作台内容区建议采用 `主区 70% + 侧栏 30%`
- 统计卡建议每行 `3` 或 `4` 个
- 常用应用宫格建议每行 `4 / 6 / 8` 个，根据场景调整

## 7.3 留白规律

- 页面之间依靠留白分区，不依靠重色块强切
- 模块标题与内容之间保持明确垂直节奏
- 不同业务块之间应保留明显间距，避免拼贴感

## 8. 适用与禁忌

## 8.1 适合丽滋卡尔的风格提炼方向

### 建议保留的部分

- 白底卡片系统
- 细描边和弱阴影
- 左导航 + 顶栏 + 内容区的稳定骨架
- 统计卡、列表块、图表块、任务块的模块化表达
- 大量使用 `16 / 20 / 24` 的间距节奏
- 蓝色主色作为核心操作色和关键数据色

### 建议弱化的部分

- 大面积蓝紫渐变
- 过于互联网平台化的运营 Banner
- 过多装饰性插画
- 过亮的蓝色发光效果
- 娱乐化或营销化标签样式

### 建议替换为丽滋语气的部分

- 主视觉从“互联网运营蓝紫”转为“医疗可信蓝 + 冷静青 + 少量金砂点缀”
- 页面氛围从“增长平台”转为“高端医疗服务后台”
- 强调信息清晰、服务流程、会员管理、预约排期、疗程记录、机构运营等业务秩序

## 8.2 不建议走的方向

- 过强互联网运营风
- 大面积炫彩渐变
- 过深重的暗黑驾驶舱
- 装饰大于信息的视觉表达
- 高饱和多色混搭

## 8.3 最终风格定义

### 风格定位

`高端医疗服务后台 / Premium Clinical Admin`

### 视觉气质

- 冷静
- 专业
- 高级
- 可信
- 有秩序
- 轻科技

### 页面表达方式

- 以卡片化承载复杂信息
- 以固定页面骨架保证一致性
- 以浅背景和低饱和层级提升高端感
- 以蓝色和青色建立医疗服务识别
- 以少量金砂色表达会员、权益、等级与高端服务

## 9. 适合前端实现的落地要点

1. 先建立稳定的后台骨架：顶栏、左导航、内容区。
2. 以工作台、列表页、详情页、驾驶舱四类模板优先落地。
3. 页面主区与辅助区建议使用 `70% / 30%` 或 `7:3、8:4` 分栏逻辑。
4. 统一使用白底卡片、浅灰蓝背景、弱阴影和较大圆角。
5. 蓝色主色负责操作和关键数据，青色负责辅助医疗服务语义，金砂色仅用于高端权益点缀。
6. 不要把参考图中的互联网平台氛围直接复刻到医疗后台，而要保留其结构优点并转译为更克制的医疗语气。

## 10. 一句话总结

这组素材最适合提炼成一套“浅色、高级、卡片化、蓝青主色、秩序清晰”的企业级 Web 后台设计语言。对于丽滋卡尔，最佳落地方向不是互联网平台感，而是基于同样的后台结构，转译为更克制、更可信、更高端的医疗服务管理后台。
